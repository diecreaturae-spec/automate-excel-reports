# Автоматизация отчётов по бережливому производству

## Зачем это понадобилось

Каждый месяц я тратила несколько часов на подготовку трёх приложений к отчёту. Суть была одна: взять список сотрудников из основного файла, сгруппировать по коду организации, а потом подставить эти цифры в три разных Excel-файла, каждый со своим форматом. Плюс ко всему нужно было удалять лишние строки-итоги, пересчитывать общую сумму и применять корпоративное форматирование (шрифт, границы, заглавные буквы в одном из столбцов).
Это была чистая механика, в которой легко ошибиться: забыть обновить ссылку, скопировать не ту ячейку, пропустить какой-то филиал. Поэтому я решила написать скрипт, который делает всё за меня.

## Что делает скрипт

Одним запуском он:
- Группирует сотрудников по коду МО и сохраняет этот справочник в отдельный лист исходного файла (также с форматированием).
- Берёт три файла-приложения (№1, №2, №3), убирает из них старые итоговые строки, добавляет новый столбец с количеством обученных сотрудников, используя справочник, и считает общий итог.
- Для Приложения №1, №2 дополнительно заливает серым четвёртый столбец, а в Приложении №3 очищает столбец с периодом после подстановки.
- Приводит всё к единому стилю: Times New Roman 12, рамки, центровка, а в третьем столбце переводит текст в заглавные буквы.

В результате я получаю три готовых отформатированных файла за пару секунд вместо прежних трёх-четырёх часов ручной работы.

## Как этим пользоваться

1. Скачайте репозиторий или скопируйте файлы.
2. Установите зависимости: pip install -r requirements.txt
3. Положите в ту же папку исходные файлы (названия должны совпадать: `Бережливое_производство.xlsx`, `Приложение №1.xlsx`, `Приложение №2.xlsx`, `Приложение №3(с кодом).xlsx`).
4. Запустите скрипт: python main.py
5. Заберите готовые файлы с суффиксом `_готово` — они уже отформатированы и содержат все нужные данные.

## Что использовала

- Python 3.9+
- Pandas — для группировок, маппинга и слияний
- OpenPyXL — для тонкой настройки внешнего вида Excel (шрифты, границы, цвета)
- Встроенные средства Python для работы с файлами

## С чем столкнулась и как решила

Главная сложность оказалась в том, что структура Приложения №3 отличалась от первых двух: там код организации хранился в столбце «Период», а название МО — в отдельном столбце, плюс итог нужно было писать не в тот же столбец, а в «Наименование МО». Пришлось сделать универсальную функцию, куда можно передать индексы столбцов и имена целевых полей. Теперь при изменении структуры достаточно поправить параметры в вызове.

Ещё один нюанс — динамическое определение столбца с длинным названием «Работников, устроенных...», чтобы не привязываться к жёсткому индексу. Скрипт сам находит его по фразе и применяет центровку.

## Что можно было бы улучшить

- Добавить логирование, чтобы видеть каждый шаг и возможные ошибки.
- Сделать конфигурационный файл (например, YAML), где будут прописаны все имена файлов и столбцов — тогда не придётся лезть в код, если что-то поменяется.
- Обернуть в телеграм-бота, чтобы запускать отчёты с телефона.
